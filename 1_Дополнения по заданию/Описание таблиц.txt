Будут созданы следующие таблицы:

NutritionAssessments - анкета пользователя + дата ее создания

Nutrients - наименование полезного вещества/Норма потребления/ед. измерения

NutrientIntakes - количество употребленных конкретным пользователем полезных веществ

Supplements и SupplementImages - справочник БАДов(Название, описание(для всплывающей формы, если таковая имеется), и ссылка на товар для фронтендщика. 
Вторая таблица - ссылки на картинки бадов для вьюшки, нужна только для соблюдения "условности".

NutritionAssessmentSupplements  - промежуточная таблица "Анкета -БАДы", 
поскольку EF-Core работает с связью многие-ко-многим весьма корявенько, 
то я всегда оставляю в таких случаях промежуточную таблицу "на расширение". 
В данном случае, если, допустим, возникнет необходимость добавить дату начала приема какого-то конкретного БАДа, 
то это можно будет сделать одной строкой кода без головной боли



Текстовое описание связей

NutritionAssessment 1 to many NutrientIntakes
У пользователя есть каждое полезное вещество из таблицы

Nutrient 1 to many NutrientIntakes
От обратного с прошлой связью. Условное "Железо" есть у каждого пользователя.

NutritionAssessment many to many Supplement
Через промежуточную таблицу NutritionAssessmentSupplements.
Отображает рекомендованные БАДы для пользователя.

Supplement 1 to many SupplementImages
Один БАД может иметь несколько изображений.
